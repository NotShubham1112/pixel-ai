# Contributing to Pixel

First off, thank you for considering contributing to Pixel! It's people like you that make Pixel such a great tool for the community.

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to see if the bug has already been reported.
* If you can't find an open issue that describes the problem, open a new one. Include a clear title and description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements
* Open a new issue and describe the enhancement you'd like to see, and why.

### Pull Requests
* Fill in the required template.
* Do not include more than one feature or bug fix in a single pull request.
* Ensure the test suite passes.
* Include new tests for new features.

## Stylecards

### Python Styleguide
We follow PEP 8. Please ensure your code is linted using `black`.

### Documentation
Ensure that any new features are documented in the `docs/` folder.

## Thank You!
Your contributions are highly valued and help make AI education more accessible for kids and students.
